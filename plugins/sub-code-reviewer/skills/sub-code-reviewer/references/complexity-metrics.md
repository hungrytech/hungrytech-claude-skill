# 복잡도 메트릭

> 코드 복잡도 측정 공식과 임계값.

---

## 순환 복잡도 (Cyclomatic Complexity)

**공식**: `M = E - N + 2P`
- E: 간선 수, N: 노드 수, P: 연결 컴포넌트 수

**간단 계산**: 분기문(if, else if, while, for, case, catch, &&, ||) 수 + 1

| 범위 | 위험도 | 테스트 필요 |
|------|--------|-----------|
| 1-10 | 낮음 | 기본 테스트 |
| 11-20 | 보통 | 분기별 테스트 |
| 21-50 | 높음 | 집중 테스트 필요 |
| 50+ | 매우 높음 | 리팩토링 필수 |

## 인지 복잡도 (Cognitive Complexity)

SonarQube 정의:
- 흐름 중단 (if, switch, for, while): +1
- 중첩 깊이: 중첩 레벨만큼 추가
- 시퀀스 브레이크 (break, continue, return 중간): +1
- 논리 연산자 혼합 (&&와 || 혼용): +1

| 범위 | 상태 |
|------|------|
| 0-5 | 우수 |
| 6-15 | 양호 |
| 16-25 | 주의 |
| 25+ | 리팩토링 필요 |

## 결합도 (Coupling)

- **Afferent Coupling (Ca)**: 이 클래스에 의존하는 클래스 수
- **Efferent Coupling (Ce)**: 이 클래스가 의존하는 클래스 수
- **Instability**: I = Ce / (Ca + Ce)
  - 0: 완전 안정 (다수가 의존)
  - 1: 완전 불안정 (다수에 의존)
