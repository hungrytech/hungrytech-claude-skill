# 스캔 프로토콜

> Phase 1: 리뷰 대상 파일을 탐색하고 베이스라인 메트릭을 수집한다.

---

## 실행 절차

### Step 1: 대상 파일 탐색

| 입력 유형 | 탐색 방법 |
|----------|----------|
| 파일 경로 | 단일 파일 직접 읽기 |
| 디렉토리 | `find` + 언어별 확장자 필터 |
| `git diff` | `git diff --name-only` |
| 미지정 | `git diff HEAD~3 --name-only` |

### Step 2: 언어 감지

```bash
scripts/detect-language.sh [target-path]
```

### Step 3: 베이스라인 메트릭 수집

```bash
scripts/measure-complexity.sh [target-path]
```

수집 메트릭:
- 파일별 라인 수, 메서드 수, 클래스 수
- 최대 메서드 길이
- 파일 크기 분포

### Step 4: 리뷰 스코프 결정

대상 파일 수에 따라 리뷰 깊이 결정:
- 1-5개: 파일별 상세 리뷰
- 6-20개: 패키지 수준 패턴 리뷰
- 20+: 핫스팟 분석 (가장 복잡한 상위 10개 집중)

## 출력

- 대상 파일 목록
- 언어 + 프레임워크 정보
- 베이스라인 복잡도 메트릭
- 리뷰 깊이 결정
