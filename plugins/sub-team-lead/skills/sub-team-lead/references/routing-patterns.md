# 라우팅 패턴

> 멀티 전문가 조율을 위한 표준 패턴.

---

## 1. 순차 파이프라인

전문가 A의 출력이 전문가 B의 입력이 되는 패턴.

**사용 조건**: 전문가 간 데이터 의존성 존재
**예시**: API 설계 → 컨트롤러 구현 → 테스트 생성 → CI 파이프라인

```
A ──결과──▶ B ──결과──▶ C
```

**주의사항**: 중간 단계 실패 시 파이프라인 전체 중단

## 2. 병렬 팬아웃

독립적인 전문가를 동시에 실행하는 패턴.

**사용 조건**: 전문가 간 의존성 없음
**예시**: 코드 리뷰 + 성능 분석 동시 실행

```
    ┌──▶ A ──┐
요청 ┤        ├──▶ 통합
    └──▶ B ──┘
```

**주의사항**: 결과 통합 시 충돌 해결 필요

## 3. 피드백 루프

설계와 리뷰를 반복하는 패턴.

**사용 조건**: 품질 기준 충족까지 반복 필요
**예시**: API 설계 → 리뷰 → 수정 → 재리뷰

```
A ──결과──▶ B ──이슈──▶ A (수정) ──▶ B (재리뷰)
```

**주의사항**: 최대 반복 횟수 설정 (기본 2회)

## 4. 에스컬레이션

전문가의 신뢰도가 낮거나 실패 시 team-lead가 재라우팅.

**사용 조건**: 전문가 결과 불충분 또는 실패
**예시**: 코드 리뷰어가 아키텍처 이슈 발견 → engineering-workflow 에스컬레이션

```
A ──실패/불충분──▶ team-lead ──재라우팅──▶ B
```
