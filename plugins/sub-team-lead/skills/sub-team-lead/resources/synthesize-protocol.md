# 결과 통합 프로토콜

> Phase 4: 전문가 결과를 수집하고 통합 보고서를 생성한다.

---

## 개요

모든 전문가의 실행이 완료되면, 결과를 수집하여 사용자에게 일관된 형태로 제공한다.

## 결과 수집

각 전문가의 `<sister-skill-result>` 메시지를 파싱:
- status: completed | partial | failed
- summary: 주요 결과 요약
- artifacts: 생성된 파일 목록
- metrics: 관련 메트릭

## 충돌 해결

전문가 간 상충되는 제안이 있는 경우:
1. 각 제안의 근거를 비교
2. 프로젝트 컨텍스트에 더 적합한 쪽 선택
3. 사용자에게 양쪽 의견 모두 제시 + 권장안 표시

## 통합 보고서 구성

`templates/synthesis-report.md` 템플릿 사용:
- 요청 요약
- 참여 전문가 목록
- 전문가별 결과
- 종합 결론
- 후속 조치 권장

## 후속 조치 제안

실행 결과를 분석하여 자연스러운 다음 단계를 제안:
- API 설계 완료 → "컨트롤러 구현을 진행할까요?"
- 코드 리뷰 완료 → "제안된 리팩토링을 적용할까요?"
- 성능 분석 완료 → "최적화를 구현할까요?"
